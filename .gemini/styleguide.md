# React + TypeScript 코드 리뷰 가이드라인

## 1. 언어 설정

- 모든 리뷰 댓글, 요약, 제안은 반드시 **한국어로 작성**합니다.
- 전문 용어는 관용적인 표현을 사용합니다.
  - 예: 렌더링, 프롭스 드릴링, 메모이제이션 등
- 설명은 전체적으로 한국어로 진행합니다.

---

## 2. 리뷰 중점 사항

### 🚀 성능 최적화

- 불필요한 리렌더링(Re-rendering) 유발 요소를 체크합니다.
- `useMemo`, `useCallback`의 적절한 사용 여부를 확인합니다.
- 의존성 배열(deps)의 정확성을 검토합니다.
- 대규모 리스트 렌더링 시:
  - 가상화(Virtualization) 적용 여부
  - `key` 설정의 적절성

---

### 📖 가독성 및 규칙

- TypeScript 타입이 명확히 정의되어 있는지 확인합니다.
  - `any` 사용 지양
- 네이밍 컨벤션 준수 여부 확인:
  - 변수 / 함수: `camelCase`
  - 인터페이스 / 타입: `PascalCase`

---

### 🛡️ 로직 오류 및 안정성

- Optional Chaining (`?.`) 사용 여부 확인
  - 런타임 에러 방지 (`Cannot read property...`)
- 비동기 처리(API 호출 등):
  - 에러 핸들링 누락 여부 체크

---

- CSS 수정 사항

### 📝 6. 커밋(Commit) 가이드
권장 커밋 메시지 형식:
- feat: 새로운 기능 추가
- fix: 버그 수정
- refactor: 코드 리팩토링
- docs: 문서 수정
- chore: 기타 작업

예시:
``` Bash
feat: optimize list rendering with memoization
fix: handle undefined API response safely
refactor: extract reusable hook for data fetching
```
